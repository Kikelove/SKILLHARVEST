    GIT Commands and Usage
  1. git config -l : returns a list of information about your git configuration including user name and email
  2. git config --global user.name "Kikelove" : To configure your user name
  3. git config --global user.email "kikeadebajo@gmail.com" : This command lets you setup the user email address you'll use in your commits
  4. git config --global credential.helper cache : Use this command to store login credentials in the cache so you don't have to type them in each time.
  5. git init : To initialize a new Git repo locally in your project root
  6. git add filename_here : This command will add a file to the staging area. Just replace filename_here with the name of the file you want to add to the staging area.
  7. git add . - To add all files in your project to the staging area, you can use a wildcard . and every file will be added for you.
  8. git add fil* : With the asterisk in the command, you can add all files starting with 'fil' in the staging area.
  9. git status : This command will show the status of the current repository including staged, unstaged, and untracked files.
  10. git commit : To commit changes in the editor in Git
  11. git commit -m "your commit message here" - To commit changes with a message in Git
  12. git commit -a -m"your commit message here" : To add and commit tracked files with a single command by using the -a and -m options.
  13. git log - This command shows the commit history for the current repository
  14. git log -p : This command shows the commit's history including all files and their changes
  15. git show commit-id : This command shows a specific commit. Replace commit-id with the id of the commit that you find in the commit log after the word commit.
  16. git log --stat : This command will cause the Git log to show some statistics about the changes in each commit, including line(s) changed and file names.
  17. git add -p : This command opens a prompt and asks if you want to stage changes or not, and includes other options.
  18. git rm filename : This command expects a commit message to explain why the file was deleted
  19. git mv oldfile newfile : This command stages the changes, then it expects a commit message
  20. Create a .gitignore file and commit it - To ignore files in Git
  21. git checkout filename : To revert unstaged changes in Git:
  22. git commit --amend : It allows you to modify and add changes to the most recent commit.
  23. git push -f : This command will force a push request. This is usually fine for pull request branches because nobody else should have cloned them
  24. git rebase branch_name_here : You can transfer completed work from one branch to another using git rebase.
  25. git push --delete origin branch_name_here : To remove a remote branch in Git
  26. git push -u origin branch_name : To push a new branch to a remote repo in Git
  27. git remote update : This lets you update the remote without merging any content into the local branches
  28. git revert HEAD : To rollback the last commit in Git
  29. git revert comit_id_here : To rollback an old commit in Git
  30. git branch branch_name : To create a new branch in Git
  31. git checkout branch_name : To switch to a newly created branch in Git
